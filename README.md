# AI-Driven Lead Qualification & Extraction Pipeline

Двухконтурная автоматизированная система мониторинга Telegram-каналов, фильтрации и ИИ-квалификации лидов на базе **Telethon (Python)**, **n8n** и **OpenAI GPT-4o-mini**.

---

## 💼 Бизнес-ценность решения
В B2B-продажах и сфере услуг ручной мониторинг профильных Telegram-чатов занимает часы рабочего времени менеджеров. При этом до 90% сообщений составляют спам, обсуждения общего характера или нецелевые запросы.

**Данное решение автоматизирует этот процесс:**
* **Экономия времени:** Менеджеры работают только с квалифицированными лидами.
* **Снижение расходов на ФОТ:** Автоматизация первичного скоринга заменяет рутинную работу ассистента.
* **Скорость реакции:** Целевой лид попадает в CRM и рабочий Telegram-чат в течение нескольких минут после публикации сообщения.

---

## ⚙️ Архитектура системы (Двухконтурный подход)

Система разделена на два независимых контура для обеспечения высокой скорости работы и экономии API-лимитов нейросети.

### Контур 1: Быстрая фильтрация (Python + Telethon)
Асинхронный сервис на Python активно опрашивает целевые чаты и выполняет первичную грубую фильтрацию:
* **Критерии отсева:** Сообщения короче 6 слов, сообщения со сторонними ссылками (спам/реклама).
* **Логика соответствия:** Проверка на пересечение ключевых слов и интентов (например, наличие слов "нужен/ищу" + "разработка/бот").
* **Исключение стоп-слов:** Быстрый пропуск нецелевых тем.
* **Динамический конфиг:** Скрипт каждые 15 минут опрашивает n8n и обновляет списки чатов, ключевых слов и стоп-слов без перезапуска сервиса.
* **Отказоустойчивость:** Реализован `connection_watchdog` для автоматического переподключения при обрыве сессии Telegram.

### Контур 2: ИИ-анализ и маршрутизация (n8n + LLM)
Отфильтрованные сообщения отправляются на Webhook n8n, где происходит глубокий анализ:
1. **Унификация данных:** Приведение метаданных сообщения к единому формату.
2. **ИИ-квалификация (GPT-4o-mini):** Нейросеть анализирует контекст по строгой системной инструкции (отсеивает новичков, запросы на базовые консультации, жалобы и предложения услуг).
3. **Формирование саммари:** ИИ извлекает суть запроса и причину квалификации.
4. **Дистрибуция:** Целевые лиды отправляются в рабочий Telegram-чат команды и параллельно логируются в Google Sheets (база для CRM).

## 🛠 Технологический стек
* **Backend:** Python 3.10+, Telethon (асинхронный клиент для Telegram API).
* **Orchestration & No-Code:** n8n (Self-hosted).
* **AI Engine:** OpenAI API (GPT-4o-mini / LangChain интеграция в n8n).
* **Infrastructure:** Systemd (для автозапуска и контроля состояния сервиса на Linux-сервере), Google Sheets API.

---

## 📂 Структура репозитория
* `parser.py` — асинхронный Python-скрипт парсера с watchdog-таймером и динамическим обновлением конфигурации.
* `parser.service` — конфигурационный файл Systemd для обеспечения отказоустойчивости (авторестарт при падении).
* `n8n/workflows/` — экспортированные схемы сценариев n8n в формате JSON:
  * `lead_qualification.json` — контур ИИ-анализа, скоринга и отправки уведомлений.
  * `config_sync.json` — контур сборки динамического конфига из Google Sheets для Python-скрипта.
* `.env.example` — шаблон переменных окружения для быстрого развертывания.

---

## 🚀 Как развернуть систему

### 1. Настройка n8n
1. Импортируйте воркфлоу из папки `n8n/workflows/` в ваш экземпляр n8n.
2. Настройте подключения (Credentials) для Google Sheets, OpenAI и Telegram Bot API.
3. Активируйте сценарии и скопируйте URL-адреса полученных вебхуков.

### 2. Настройка Python-скрипта
1. Клонируйте репозиторий на сервер.
2. Создайте файл `.env` на основе `.env.example` и заполните его вашими данными (API ID/Hash от Telegram, URL вебхуков n8n).
3. Установите зависимости:
   ```bash
   pip install telethon requests urllib3
